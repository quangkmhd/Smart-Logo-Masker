# Configuration Guide for Smart Logo Masker

Smart Logo Masker relies on a combination of environment files, YAML configurations, and command-line flags to dictate its behavior across training and inference.

## 1. Environment Configuration (`.env`)

The project uses an `.env` file to manage global settings and Docker configurations. Copy `.env.example` to `.env` to begin.

| Variable | Default Value | Description |
|----------|---------------|-------------|
| `HOST_PORT` | `8080` | The external port exposed by Docker if running the web UI/API container. |
| `NVIDIA_VISIBLE_DEVICES` | `all` | Controls which GPUs the Docker container can access. Change to `0` to restrict to the first GPU. |
| `LOG_LEVEL` | `INFO` | Standard logging verbosity. Set to `DEBUG` when diagnosing inference pipeline crashes. |

## 2. YOLO Dataset Configuration (`data.yaml`)

When training the model, YOLO requires a specific YAML configuration file. This file is automatically generated by `prepare_data.py`, but you may need to edit it manually if adding new classes.

**Example `data.yaml`:**
```yaml
train: /workspace/dataset/images/train/
val: /workspace/dataset/images/val/

# Number of classes
nc: 2

# Class names
names:
  0: 1xbet
  1: melbet
```
*Critical Note:* If you manually edit the `names` array, you must ensure the indices (`0`, `1`) exactly match the integer class IDs present in your YOLO label `.txt` files.

## 3. Docker Configuration (`docker-compose.yml`)

The `docker-compose.yml` file is pre-configured for GPU acceleration.

### GPU Passthrough
To ensure the container utilizes your NVIDIA GPU, the compose file includes the `deploy.resources.reservations.devices` block.
```yaml
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: 1
              capabilities: [gpu]
```
If you only have a CPU, you must comment out this block and rebuild the image, otherwise `docker-compose up` will fail complaining about the missing NVIDIA runtime.

### Volume Mapping
```yaml
    volumes:
      - ../asset:/workspace/asset
      - ../runs:/workspace/runs
```
This ensures that any videos you place in the local `asset/` folder are immediately visible inside the container at `/workspace/asset/`, and inference outputs saved to `/workspace/runs/` persist on your local drive even after the container shuts down.

## 4. Hyperparameter Tuning for Inference

When executing `main.py`, getting the perfect mask requires balancing a few parameters:

- **`--conf-thres` (Confidence Threshold)**:
  - If set too low (e.g., `0.1`): The model will aggressively blur things that look vaguely like logos (text on scoreboards, generic sponsor patches).
  - If set too high (e.g., `0.8`): The model will fail to blur the target logo when it is motion-blurred, rotated, or partially occluded by a player's arm.
  - *Recommended:* Start at `0.4` and adjust based on visual feedback.

- **`--blur-kernel`**:
  - Dictates the intensity of the blur. 
  - Must be an odd integer (e.g., 51, 99, 151).
  - For 1080p video, `99` provides a strong blur that completely obfuscates readable text. For 4K video, you may need to increase this to `151` to achieve the same visual obfuscation effect.
