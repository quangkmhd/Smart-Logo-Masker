# Smart Logo Masker API & Code Reference

This document outlines the command-line interfaces and internal scripts used to operate the Smart Logo Masker.

## 1. CLI Commands

### 1.1. Inference (`main.py`)

The primary script for running the logo masking pipeline on videos or camera feeds.

```bash
python3 main.py --source <path> --weights <path> [OPTIONS]
```

**Arguments:**
- `--source` (str, required): The input path. Can be:
  - A path to a specific video file (e.g., `./asset/test.mp4`).
  - A path to a directory containing multiple videos (e.g., `./input_folder/`).
  - An integer (e.g., `0`) to specify a local webcam device ID.
- `--weights` (str, required): Path to the PyTorch checkpoint (`.pt` file) containing the trained YOLO segmentation model.
- `--conf-thres` (float, optional): Confidence threshold for detection. Default: `0.25`. Increase this if the model is blurring things that aren't logos (false positives).
- `--iou-thres` (float, optional): NMS IoU threshold. Default: `0.45`. Controls overlapping detection resolution.
- `--imgsz` (int, optional): Inference image size. Default: `640`. Increase to `1280` for 4K video to detect smaller logos.
- `--save-dir` (str, optional): Directory to save the masked output videos. Default: `./runs/predict/`.
- `--show` (flag): If passed, displays the video with applied masks in a real-time OpenCV window during processing.
- `--blur-kernel` (int, optional): The intensity of the Gaussian blur applied to the logo. Must be an odd number. Default: `99`.

### 1.2. Training (`train_yolo.py` / `train/train.py`)

Initiates the fine-tuning process for the segmentation model.

```bash
python3 train_yolo.py --data <path> --epochs <int> [OPTIONS]
```

**Arguments:**
- `--data` (str, required): Path to the `data.yaml` file generated by the data preparation script.
- `--weights` (str, optional): Path to initial weights for transfer learning. Default: `yolov26-seg.pt` (downloads automatically if missing).
- `--epochs` (int, required): Number of training epochs.
- `--batch` (int, optional): Batch size. Default: `16`. Reduce if experiencing CUDA Out of Memory errors.
- `--device` (str, optional): CUDA device (e.g., `0` or `0,1,2,3`). Use `cpu` to force CPU training.
- `--patience` (int, optional): Early stopping patience. Default: `50`.

### 1.3. Data Preparation (`prepare_data.py`)

Converts LabelMe JSON annotations into YOLO segmentation format.

```bash
python3 prepare_data.py --input <path> --output <path> --split <float>
```

**Arguments:**
- `--input` (str, optional): Path to the directory containing raw images and `.json` files. Default: `./data_raw/`.
- `--output` (str, optional): Path where the YOLO-formatted dataset will be created. Default: `./dataset/`.
- `--split` (float, optional): The percentage of data to allocate to the training set (the rest goes to validation). Default: `0.8` (80% train, 20% val).

### Mask Application Logic Snippet

Inside the inference loop, the mask is applied using OpenCV. Developers can modify this logic to change the censor style (e.g., black box instead of blur).

```python
# Extracted from inference loop
for det in detections:
    mask = det.mask # Binary polygon array
    
    # 1. Extract ROI
    roi = frame[mask]
    
    # 2. Apply Censor Effect (Gaussian Blur)
    # Developers can replace this with: roi[:] = (0,0,0) for a black box
    blurred_roi = cv2.GaussianBlur(roi, (k_size, k_size), 0)
    
    # 3. Composite back
    frame[mask] = blurred_roi
```
